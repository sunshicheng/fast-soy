### 整体规划
1. 清洗真实的数据 CMB-Clin-summary.xlsx 到数据库，记录疾病和症状
2. 扮演一个患病的患者，结合数据库中的症状。根据用户的输入疾病进行获取症状
3. 调用医生接口，根据医生的问题，抽取症状进行回复，没有的答案根据情况推测一个写上
4. 获取医生接口的结果，然后进行疾病比对，确认是不是用户输入的疾病。包含也可以。
5. agent 规划
    - 主控agent 用于规划下一步需要做的操作
    - 生成首次对话问候语的agent 
    - 疾病症状总结agent 用于分析疾病的症状
    - 对话审查agent 用于结合上下文内容，对疾病症状总结内容进行审查和改写
    - 结果判断agent 用于对话处理完成之后，提取医生的回复疾病和用户给出的疾病是否一致
6. 技术实现要求
    - python +langchain + langgraph + postgresql
    - 对话维护一个session postgresql 数据库中
    - 用户token postgresql 中，过期了之后再更新

    - 调用医生接口交互.md 用于和医生交互分为深度和非深度
